# Red-Black Tree (RBT) Learning Order

Red-Black Trees are the next step after AVL Trees.

```text
Binary Tree
↓
BST
↓
AVL Tree
↓
Red-Black Tree
↓
B-Tree
↓
B+ Tree
```

Red-Black Trees are widely used in:

* C++ STL (`std::map`, `std::set`)
* Java Collections Framework (`TreeMap`, `TreeSet`)
* Linux Kernel
* OpenJDK

# Phase 2: Red-Black Tree Properties

Learn the 5 rules:

### Rule 1

```text
Every node is either Red or Black
```

### Rule 2

```text
Root is Black
```

### Rule 3

```text
NULL leaves are Black
```

### Rule 4

```text
No two adjacent Red nodes
```

### Rule 5

```text
Every path from a node to a NULL leaf
contains the same number of Black nodes
```

This count is called:

```text
Black Height
```

# Phase 3: Rotations

Same rotations as AVL.

### LL Case

```text
Right Rotation
```

### RR Case

```text
Left Rotation
```

### LR Case

```text
Left Rotation
then
Right Rotation
```

### RL Case

```text
Right Rotation
then
Left Rotation
```

# Phase 4: Red-Black Tree Insertion

Learning order:

```text
BST Insert
↓
Insert New Node as Red
↓
Check Violations
↓
Recolor
↓
Rotate
```

---

## Case 1

```text
Parent Black
```

No action.

---

## Case 2

```text
Parent Red
Uncle Red
```

Recolor.

---

## Case 3

```text
Parent Red
Uncle Black
```

Rotate + Recolor.

---

# Phase 5: Red-Black Tree Deletion

Learning order:

```text
BST Delete
↓
Fix Double Black
↓
Recolor
↓
Rotate
```

Deletion is considerably harder than insertion.

# AVL vs Red-Black Tree

| Feature | AVL            | Red-Black          |
| ------- | -------------- | ------------------ |
| Balance | Strict         | Relaxed            |
| Height  | Smaller        | Slightly Larger    |
| Search  | Faster         | Slightly Slower    |
| Insert  | More Rotations | Fewer Rotations    |
| Delete  | More Rotations | Fewer Rotations    |
| Used In | Academic       | Production Systems |

### Most Important Problem

Balance a Binary Search Tree

Although not a Red-Black Tree problem, it teaches the core goal of self-balancing trees: maintaining approximately **O(log n)** height for efficient search, insertion, and deletion.
