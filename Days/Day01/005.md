# Problem: Palindrome Number

Check whether an integer reads the same forward and backward.

**Asked By:** Amazon, Google, Microsoft, Apple
**Difficulty:** Easy

---

# Solution Patterns

## Pattern 1: Convert to String

### Idea

Convert the number to a string and compare it with its reverse.

Example:

```text
121
"121" == "121"
```

### Complexity

* Time: O(n)
* Space: O(n)

where n = number of digits.

### Code

```python
def isPalindrome(x):
    if x < 0:
        return False

    s = str(x)
    return s == s[::-1]
```

---

## Pattern 2: Two Pointer on String

### Idea

Convert number to string.

Compare leftmost and rightmost digits moving inward.

### Complexity

* Time: O(n)
* Space: O(n)

### Code

```python
def isPalindrome(x):
    if x < 0:
        return False

    s = str(x)

    left = 0
    right = len(s) - 1

    while left < right:
        if s[left] != s[right]:
            return False

        left += 1
        right -= 1

    return True
```

---

## Pattern 3: Reverse Entire Number

### Idea

Reverse the number mathematically and compare with original.

Example:

```text
121

reverse = 121

121 == 121
```

### Complexity

* Time: O(log₁₀ n)
* Space: O(1)

### Code

```python
def isPalindrome(x):
    if x < 0:
        return False

    original = x
    reverse = 0

    while x > 0:
        digit = x % 10
        reverse = reverse * 10 + digit
        x //= 10

    return original == reverse
```

---

## Pattern 4: Reverse Half of Number (Optimal)

### Idea

No string conversion.

Reverse only half the digits.

Observations:

* Negative numbers are never palindromes.
* Numbers ending with 0 are not palindromes unless the number itself is 0.

Example:

```text
1221

x       = 12
reverse = 12

Palindrome
```

### Complexity

* Time: O(log₁₀ n)
* Space: O(1)

### Code

```python
def isPalindrome(x):
    if x < 0 or (x % 10 == 0 and x != 0):
        return False

    reverse_half = 0

    while x > reverse_half:
        reverse_half = reverse_half * 10 + x % 10
        x //= 10

    return x == reverse_half or x == reverse_half // 10
```

# Pattern Comparison

| Pattern               | Time     | Space | String Used | Interview Rating |
| --------------------- | -------- | ----- | ----------- | ---------------- |
| String Reverse        | O(n)     | O(n)  | Yes         | Basic            |
| Two Pointer String    | O(n)     | O(n)  | Yes         | Basic            |
| Reverse Entire Number | O(log n) | O(1)  | No          | Good             |
| Reverse Half Number   | O(log n) | O(1)  | No          | Best             |

---

# Why Reverse Half is Better

Consider:

```text
123454321
```

Full Reverse:

```text
Reverse all 9 digits
```

Half Reverse:

```text
Reverse only 4-5 digits
```

Benefits:

* Less work
* Constant space
* No string conversion
* Avoids unnecessary digit processing
* Standard interview solution
