# Count Primes

**Problem Statement:**
Count the number of prime numbers strictly less than `n`.

**Asked By:** Amazon, Microsoft, Google, LinkedIn
**Difficulty:** Medium

---

# Solution Patterns

## Pattern 1: Brute Force Prime Checking

### Idea

For every number from `2` to `n-1`, check whether it is prime.

### Algorithm

1. Iterate from `2` to `n-1`.
2. For each number, check divisibility from `2` to `num-1`.
3. Count primes.

### Complexity

* Time: **O(n²)**
* Space: **O(1)**

### Code

```python
def countPrimes(n):
    count = 0

    for num in range(2, n):
        prime = True

        for i in range(2, num):
            if num % i == 0:
                prime = False
                break

        if prime:
            count += 1

    return count
```

---

## Pattern 2: Prime Checking up to √n

### Idea

A number cannot have a factor greater than its square root unless a smaller factor already exists.

### Algorithm

1. For each number from `2` to `n-1`.
2. Check divisibility only up to `√num`.
3. Count valid primes.

### Complexity

* Time: **O(n√n)**
* Space: **O(1)**

### Code

```python
def countPrimes(n):
    def is_prime(num):
        if num < 2:
            return False

        i = 2
        while i * i <= num:
            if num % i == 0:
                return False
            i += 1

        return True

    count = 0

    for num in range(2, n):
        if is_prime(num):
            count += 1

    return count
```

## Pattern 3: Basic Sieve of Eratosthenes

### Idea

Instead of checking every number individually, mark multiples of each prime as non-prime.

### Algorithm

1. Create an array initialized to True.
2. Start from 2.
3. Mark all multiples of every prime as False.
4. Count remaining True values.

### Complexity

* Time: **O(n log log n)**
* Space: **O(n)**

### Code

```python
def countPrimes(n):
    if n <= 2:
        return 0

    prime = [True] * n
    prime[0] = prime[1] = False

    for i in range(2, n):
        if prime[i]:
            multiple = i * 2

            while multiple < n:
                prime[multiple] = False
                multiple += i

    return sum(prime)
```

## Pattern 4: Optimized Sieve of Eratosthenes

### Idea

Two optimizations:

1. Only iterate until `√n`.
2. Start marking from `i²` because smaller multiples were already processed.

### Algorithm

1. Create sieve array.
2. Iterate from `2` to `√n`.
3. If prime:

   * Mark multiples from `i²`.
4. Count remaining primes.

### Complexity

* Time: **O(n log log n)**
* Space: **O(n)**

### Code

```python
def countPrimes(n):
    if n <= 2:
        return 0

    prime = [True] * n
    prime[0] = prime[1] = False

    i = 2

    while i * i < n:
        if prime[i]:
            multiple = i * i

            while multiple < n:
                prime[multiple] = False
                multiple += i

        i += 1

    return sum(prime)
```

## Pattern 5: Pythonic Optimized Sieve (Interview + Production)

### Idea

Use slice assignment to mark multiples in bulk.

### Complexity

* Time: **O(n log log n)**
* Space: **O(n)**

### Code

```python
def countPrimes(n):
    if n <= 2:
        return 0

    prime = [True] * n
    prime[0] = prime[1] = False

    limit = int(n ** 0.5) # sqrt(n)
    for i in range(2, limit + 1):
        if prime[i]:
            prime[i * i:n:i] = [False] * len(prime[i * i:n:i])

    return sum(prime)
```

# Optimized Solution

For interviews, the expected solution is the **Optimized Sieve of Eratosthenes**.

```python
def countPrimes(n):
    if n <= 2:
        return 0

    prime = [True] * n
    prime[0] = prime[1] = False

    for i in range(2, int(n ** 0.5) + 1):
        if prime[i]:
            for j in range(i * i, n, i):
                prime[j] = False

    return sum(prime)
```

### Why this is optimal?

* Eliminates repeated primality checks.
* Starts marking from `i²`.
* Processes each composite number very few times.
* Standard accepted solution in Amazon, Google, Microsoft, and LinkedIn interviews.
* Time Complexity: **O(n log log n)**
* Space Complexity: **O(n)**
