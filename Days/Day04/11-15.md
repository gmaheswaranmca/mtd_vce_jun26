# Questions: (11-15)
1. Subsequence Check, Write a program that checks if string T is a subsequence of string S. A subsequence maintains character order but allows skipping characters.
2. Find All Unique Pairs with Sum K, You are given a list of integers provided as a comma-separated string and an integer K. Write a program to find all unique pairs of numbers from the list such that the sum of each pair is equal to K. Each valid pair should be considered unique (i.e., (a, b) and (b, a) are the same), printed in ascending order within the pair, and printed one pair per line as a tuple. For example, if the input is 5,3,7,9,5 with K = 12, valid unique pairs whose sum is 12 are (3, 9) and (5, 7).
3. Word Reversal Transform, You are building a text-transformation utility for a document-processing pipeline. Given a sentence composed of words separated by single spaces, reverse the characters of every individual word while keeping the words in their original order. The sentence contains only lowercase English letters and spaces.
4. Snowmelt Collection Between Mountain Peaks, You are given an array of n non-negative integers representing the height profile of a mountain range. Each column has a width of 1 unit. After snowmelt, water flows downward and gets trapped between taller peaks on either side. For each position, the amount of water it can hold is determined by the minimum of the tallest peak to its left and the tallest peak to its right, minus its own height (if positive).
5. Conveyor Belt Order Reversal, In an automated warehouse, a conveyor belt carries N packages sequentially. Due to a routing change, the packages must be dispatched in the reverse order from which they arrived.
Your task is to reverse the sequence of package IDs in-place using two pointer movement only — no extra array should be used to store the reversed result.

# 1. Subsequence Check

## Pattern Used

**Two Pointers**

### Why?

* Characters must appear in the same order.
* We can scan both strings once.
* No need for extra storage.

### Pseudocode

```text
INPUT S, T

i ← 0        // pointer for S
j ← 0        // pointer for T

WHILE i < length(S) AND j < length(T)

    IF S[i] = T[j]
        j ← j + 1

    i ← i + 1

IF j = length(T)
    PRINT "YES"
ELSE
    PRINT "NO"
```

### Complexity

| Metric | Value |
| ------ | ----- |
| Time   | O(n)  |
| Space  | O(1)  |

### Recognition Clues

* "Check whether sequence exists in order"
* "Order matters"
* "Skipping allowed"

# 2. Find All Unique Pairs with Sum K

## Possible Patterns

### Pattern 1: Brute Force

Check every pair.

```text
FOR i = 0 TO n-1
    FOR j = i+1 TO n-1
        IF arr[i] + arr[j] = K
            PRINT pair
```

Time: O(n²)

---

### Pattern 2: Hash Set (Best)

### Why?

* One pass solution.
* Handles duplicate values efficiently.
* Easy to ensure uniqueness.

### Pseudocode

```text
INPUT array, K

seen ← empty set
pairs ← empty set

FOR each num in array

    complement ← K - num

    IF complement exists in seen

        a ← minimum(num, complement)
        b ← maximum(num, complement)

        ADD (a,b) into pairs

    ADD num into seen

FOR each pair in pairs sorted
    PRINT pair
```

### Complexity

| Metric | Value |
| ------ | ----- |
| Time   | O(n)  |
| Space  | O(n)  |

### Recognition Clues

* Pair sum
* Target value
* Unique pairs
* Fast lookup needed

# 3. Word Reversal Transform

## Pattern Used

**String Traversal + Builder**

### Why?

Each word is independent.

### Example

```text
hello world

olleh dlrow
```

### Pseudocode

```text
INPUT sentence

words ← split(sentence)

FOR each word in words

    reverse_word ← reverse(word)

    append reverse_word to result

PRINT result joined with spaces
```

### Manual Reverse Version

```text
FOR each word

    reversed ← ""

    FOR i from length(word)-1 DOWNTO 0
        reversed ← reversed + word[i]
```

### Complexity

| Metric | Value |
| ------ | ----- |
| Time   | O(n)  |
| Space  | O(n)  |

### Recognition Clues

* Reverse each word
* Keep word order unchanged

# 4. Snowmelt Collection Between Mountain Peaks

(Equivalent to Trapping Rain Water)

## Possible Patterns

### Pattern 1: Brute Force

For each position:

```text
Find max left
Find max right

water =
MIN(max_left,max_right)-height[i]
```

Time: O(n²)

---

### Pattern 2: Prefix/Suffix Arrays
```text
leftMax[i]
rightMax[i]
```

Time: O(n)

Space: O(n)

```
INPUT height[]

n ← length(height)

CREATE leftMax[n]
CREATE rightMax[n]

leftMax[0] ← height[0]

FOR i ← 1 TO n-1

    leftMax[i] ←
        MAX(leftMax[i-1], height[i])

rightMax[n-1] ← height[n-1]

FOR i ← n-2 DOWNTO 0

    rightMax[i] ←
        MAX(rightMax[i+1], height[i])

water ← 0

FOR i ← 0 TO n-1

    trapped ←
        MIN(leftMax[i], rightMax[i])
        - height[i]

    water ← water + trapped

PRINT water
```



---

### Pattern 3: Two Pointers (Best)

### Why?

Avoids extra arrays.

### Pseudocode

```text
left ← 0
right ← n-1

leftMax ← 0
rightMax ← 0

water ← 0

WHILE left < right

    IF height[left] < height[right]

        IF height[left] >= leftMax
            leftMax ← height[left]
        ELSE
            water ← water + (leftMax - height[left])

        left ← left + 1

    ELSE

        IF height[right] >= rightMax
            rightMax ← height[right]
        ELSE
            water ← water + (rightMax - height[right])

        right ← right - 1

PRINT water
```

### Complexity

| Metric | Value |
| ------ | ----- |
| Time   | O(n)  |
| Space  | O(1)  |

### Recognition Clues

* Water trapped
* Left boundary + right boundary
* Height array

# 5. Conveyor Belt Order Reversal

## Pattern Used

**Two Pointers + Swap**

### Why?

* Reverse in-place
* No extra array allowed

### Example

```text
1 2 3 4 5

5 4 3 2 1
```

### Pseudocode

```text
left ← 0
right ← n-1

WHILE left < right

    SWAP arr[left] and arr[right]

    left ← left + 1
    right ← right - 1

PRINT array
```

### Complexity

| Metric | Value |
| ------ | ----- |
| Time   | O(n)  |
| Space  | O(1)  |

### Recognition Clues

* Reverse array
* In-place
* No extra memory

# Quick Interview Pattern Mapping

| Interview Clue                     | Pattern                    |
| ---------------------------------- | -------------------------- |
| Order maintained, skipping allowed | Two Pointers (Subsequence) |
| Find pair with target sum          | Hash Set / Two Pointers    |
| Reverse every word                 | String Traversal           |
| Water trapped between bars         | Two Pointers               |
| Reverse array in-place             | Two Pointers + Swap        |

These five problems are excellent representatives of four highly recurring interview patterns:

1. **Two Pointers** → Subsequence Check, Snowmelt Collection, Conveyor Belt Reversal
2. **Hashing** → Unique Pair Sum K
3. **String Traversal** → Word Reversal Transform
4. **Boundary Tracking (Left/Right Max)** → Snowmelt Collection (specialized two-pointer pattern)
