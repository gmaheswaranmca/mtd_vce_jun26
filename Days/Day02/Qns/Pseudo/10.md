# Minimum Window Substring

**Problem:** Find the smallest substring in `s` that contains all characters of `t` (including frequencies).

**Companies:** Google, Meta, Amazon, Microsoft, Bloomberg

**Difficulty:** Hard

---

# Solution Patterns

## Pattern 1: Brute Force (Generate All Substrings)

### Idea

Generate every possible substring and check whether it contains all characters of `t`.

### Time Complexity

* O(N³)

### Space Complexity

* O(K)

Where:

* N = length of string `s`
* K = unique characters

### Pseudocode

```text
FUNCTION MinimumWindow(s, t)

    answer = ""

    FOR start = 0 TO LENGTH(s)-1

        FOR end = start TO LENGTH(s)-1

            substring = s[start...end]

            IF ContainsAllCharacters(substring, t)

                IF answer IS EMPTY OR
                   LENGTH(substring) < LENGTH(answer)

                    answer = substring

    RETURN answer
```

---

# Pattern 2: Improved Brute Force with Frequency Count

### Idea

Expand the window and maintain frequencies instead of rebuilding counts repeatedly.

### Time Complexity

* O(N²)

### Space Complexity

* O(K)

### Pseudocode

```text
FUNCTION MinimumWindow(s, t)

    answer = ""

    targetFreq = BuildFrequency(t)

    FOR start = 0 TO LENGTH(s)-1

        currentFreq = EMPTY_MAP

        FOR end = start TO LENGTH(s)-1

            currentFreq[s[end]]++

            IF Covers(currentFreq, targetFreq)

                candidate = s[start...end]

                IF answer IS EMPTY OR
                   LENGTH(candidate) < LENGTH(answer)

                    answer = candidate

                BREAK

    RETURN answer
```

# Pattern 3: Sliding Window + Frequency Maps (Optimal)

### Idea

Maintain a sliding window.

1. Expand right pointer until all required characters are included.
2. Shrink left pointer while still satisfying requirements.
3. Track the smallest valid window.

This is the standard interview solution.

### Time Complexity

* O(N)

### Space Complexity

* O(K)

### Pseudocode

```text
FUNCTION MinimumWindow(s, t)

    IF LENGTH(t) > LENGTH(s)
        RETURN ""

    targetFreq = EMPTY_MAP

    FOR EACH ch IN t
        targetFreq[ch]++

    required = SIZE(targetFreq)

    formed = 0

    windowFreq = EMPTY_MAP

    left = 0

    bestLength = ∞
    bestStart = 0

    FOR right = 0 TO LENGTH(s)-1

        currentChar = s[right]

        windowFreq[currentChar]++

        IF currentChar IN targetFreq AND
           windowFreq[currentChar] =
           targetFreq[currentChar]

            formed++

        WHILE left <= right AND
              formed = required

            IF right - left + 1 < bestLength

                bestLength = right - left + 1
                bestStart = left

            leftChar = s[left]

            windowFreq[leftChar]--

            IF leftChar IN targetFreq AND
               windowFreq[leftChar] <
               targetFreq[leftChar]

                formed--

            left++

    IF bestLength = ∞
        RETURN ""

    RETURN SUBSTRING(s,
                     bestStart,
                     bestLength)
```

# Pattern 4: Sliding Window with Fixed Array

### Idea

When input is limited to ASCII characters:

* Use array indexing instead of hash maps.
* Faster constant factors.
* Same algorithm as Pattern 3.

### Time Complexity

* O(N)

### Space Complexity

* O(1)

(Fixed-size character set)

### Pseudocode

```text
FUNCTION MinimumWindow(s, t)

    target[128] = {0}
    window[128] = {0}

    FOR EACH ch IN t
        target[ASCII(ch)]++

    requiredCount = LENGTH(t)

    left = 0
    minLength = ∞
    startIndex = 0

    FOR right = 0 TO LENGTH(s)-1

        current = ASCII(s[right])

        window[current]++

        IF target[current] > 0 AND
           window[current] <= target[current]

            requiredCount--

        WHILE requiredCount = 0

            IF right-left+1 < minLength

                minLength = right-left+1
                startIndex = left

            leftChar = ASCII(s[left])

            window[leftChar]--

            IF target[leftChar] > 0 AND
               window[leftChar] <
               target[leftChar]

                requiredCount++

            left++

    IF minLength = ∞
        RETURN ""

    RETURN SUBSTRING(s,
                     startIndex,
                     minLength)
```

# Pattern Comparison

| Pattern                   | Approach                  | Time  | Space |
| ------------------------- | ------------------------- | ----- | ----- |
| Brute Force               | Generate all substrings   | O(N³) | O(K)  |
| Improved Brute Force      | Running frequencies       | O(N²) | O(K)  |
| Sliding Window + Hash Map | Expand/Shrink dynamically | O(N)  | O(K)  |
| Sliding Window + Array    | ASCII optimization        | O(N)  | O(1)  |

# Optimized Solution

## Interview-Preferred Solution

### Pattern

**Sliding Window + Frequency Maps**

### Why Optimal?

* Each character enters the window once.
* Each character leaves the window once.
* No redundant substring generation.
* Standard solution expected by interviewers.

### Complexity

```text
Time  : O(N)
Space : O(K)
```

Where:

* N = length of source string
* K = unique characters in target string

---

## Fastest Practical Variant

### Pattern

**Sliding Window + Fixed Array**

### Complexity

```text
Time  : O(N)
Space : O(1)
```

### Advantage

* Faster than hash maps.
* Ideal when character set is known (ASCII).

# Pattern Recognition

Use the **Sliding Window Pattern** when you see:

* Smallest valid substring
* Longest valid substring
* Continuous segment constraints
* Character frequency requirements
* At least / exactly K occurrences
* Window expansion and contraction

Common problems:

* Minimum Window Substring
* Longest Substring Without Repeating Characters
* Find All Anagrams in a String
* Permutation in String
* Longest Repeating Character Replacement
* Minimum Size Subarray Sum

### Recommended Interview Answer

```text
Pattern : Sliding Window + Frequency Map
Time    : O(N)
Space   : O(K)
```

This is the standard optimal solution for **Minimum Window Substring** and one of the most important applications of the **Sliding Window** pattern.
