# Longest Substring Without Repeating Characters

**Problem:** Find the length of the longest substring that contains only unique (non-repeating) characters.

**Companies:** Amazon, Google, Meta, Microsoft, Apple

**Difficulty:** Medium

---

# Solution Patterns

## Pattern 1: Brute Force (Generate All Substrings)

### Idea

Generate every substring and check whether all characters are unique.

### Time Complexity

* O(N³)

### Space Complexity

* O(N)

### Pseudocode

```text
FUNCTION LongestUniqueSubstring(s)

    maxLength = 0

    FOR start = 0 TO LENGTH(s)-1

        FOR end = start TO LENGTH(s)-1

            substring = s[start...end]

            IF AllCharactersUnique(substring)

                maxLength = MAX(maxLength,
                                LENGTH(substring))

    RETURN maxLength


FUNCTION AllCharactersUnique(str)

    seen = EMPTY_SET

    FOR EACH ch IN str

        IF ch IN seen
            RETURN FALSE

        INSERT ch INTO seen

    RETURN TRUE
```

---

## Pattern 2: Sliding Window with Set

### Idea

Maintain a window containing only unique characters.

When a duplicate appears:

* Shrink the window from the left
* Remove characters until duplicate disappears

### Time Complexity

* O(2N) ≈ O(N)

### Space Complexity

* O(K)

K = unique characters

### Pseudocode

```text
FUNCTION LongestUniqueSubstring(s)

    left = 0
    maxLength = 0

    seen = EMPTY_SET

    FOR right = 0 TO LENGTH(s)-1

        WHILE s[right] IN seen

            REMOVE s[left] FROM seen
            left++

        INSERT s[right] INTO seen

        maxLength = MAX(maxLength,
                        right - left + 1)

    RETURN maxLength
```

---

## Pattern 3: Sliding Window with Hash Map

### Idea

Store the last position of each character.

When a duplicate appears:

* Jump the left pointer directly
* Avoid removing one character at a time

### Time Complexity

* O(N)

### Space Complexity

* O(K)

### Pseudocode

```text
FUNCTION LongestUniqueSubstring(s)

    lastSeen = EMPTY_MAP

    left = 0
    maxLength = 0

    FOR right = 0 TO LENGTH(s)-1

        currentChar = s[right]

        IF currentChar IN lastSeen

            left = MAX(left,
                       lastSeen[currentChar] + 1)

        lastSeen[currentChar] = right

        maxLength = MAX(maxLength,
                        right - left + 1)

    RETURN maxLength
```

---

## Pattern 4: Fixed Character Set Optimization

### Idea

For ASCII-only inputs, use an array instead of a hash map.

Store the last occurrence index of each character.

### Time Complexity

* O(N)

### Space Complexity

* O(1)

(Fixed ASCII size)

### Pseudocode

```text
FUNCTION LongestUniqueSubstring(s)

    index[256]

    FILL index WITH -1

    left = 0
    maxLength = 0

    FOR right = 0 TO LENGTH(s)-1

        ascii = ASCII_VALUE(s[right])

        left = MAX(left,
                   index[ascii] + 1)

        index[ascii] = right

        maxLength = MAX(maxLength,
                        right - left + 1)

    RETURN maxLength
```

# Pattern Comparison

| Pattern                   | Approach                   | Time  | Space |
| ------------------------- | -------------------------- | ----- | ----- |
| Brute Force               | Generate all substrings    | O(N³) | O(N)  |
| Sliding Window + Set      | Expand/Shrink window       | O(N)  | O(K)  |
| Sliding Window + Hash Map | Jump left pointer directly | O(N)  | O(K)  |
| Fixed Array               | Array lookup for ASCII     | O(N)  | O(1)  |

---

# Optimized Solution

## Interview-Preferred Solution

### Pattern

**Sliding Window + Hash Map**

### Why Optimal?

* Single pass through the string.
* Directly jumps over duplicates.
* Works for any character set.
* Most frequently expected by interviewers.

### Complexity

```text
Time  : O(N)
Space : O(K)
```

---

## Most Efficient for ASCII Inputs

### Pattern

**Sliding Window + Fixed Array**

### Complexity

```text
Time  : O(N)
Space : O(1)
```

### Advantage

* Faster than hash maps due to array indexing.
* Common optimization in high-performance implementations.

---

# Pattern Recognition

Use the **Sliding Window Pattern** when you see:

* Longest substring
* Shortest substring
* Continuous segment
* Unique characters
* At most K distinct characters
* No repeating elements
* Window expansion and contraction

Typical problems:

* Longest Substring Without Repeating Characters
* Longest Repeating Character Replacement
* Minimum Window Substring
* Permutation in String
* Find All Anagrams in a String
* Longest Substring with At Most K Distinct Characters

### Recommended Interview Answer

```text
Pattern : Sliding Window + Hash Map
Time    : O(N)
Space   : O(K)
```

This is the standard optimal solution for **Longest Substring Without Repeating Characters**.
