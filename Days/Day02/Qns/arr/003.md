# Contains Duplicate

Check whether an array contains duplicate elements.

Example:

```text
Input: [1,2,3,1]
Output: True
```

```text
Input: [1,2,3,4]
Output: False
```

---

# Common Solution Patterns

## 1. Brute Force Pattern

### Idea

Compare every element with every other element.

### Pattern Used

* Nested Loop
* Comparison Pattern

### Time Complexity

* O(n²)

### Space Complexity

* O(1)

```python
    # Code Here


print(contains_duplicate([1,2,3,1]))
```

# 2. Sorting Pattern

### Idea

If duplicates exist, after sorting they become adjacent.

### Pattern Used

* Sorting
* Adjacent Comparison

### Time Complexity

* O(n log n)

### Space Complexity

* O(1) or O(n) depending on sorting implementation

### Code

```python

    # Code Here


print(contains_duplicate([1,2,3,1]))
```

# 3. Hash Set Pattern (Optimized)

### Idea

Store visited elements in a set.

If element already exists in set → duplicate found.

### Pattern Used

* Hashing
* Lookup Optimization

### Time Complexity

* O(n)

### Space Complexity

* O(n)

# Pattern Comparison

| Pattern              | Time       | Space | Interview Usage                | Recommended |
| -------------------- | ---------- | ----- | ------------------------------ | ----------- |
| Brute Force          | O(n²)      | O(1)  | Basic understanding            | ❌           |

# Interview Explanation

> “I used the Hash Set pattern because set lookup is O(1).
> While traversing the array, if an element already exists in the set, duplicate is found immediately.
> This gives O(n) time complexity with O(n) extra space.”
