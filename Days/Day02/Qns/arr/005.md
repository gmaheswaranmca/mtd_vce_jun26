# Maximum Subarray

Find the contiguous subarray with the largest sum.

Example:

```text
Input:  [-2,1,-3,4,-1,2,1,-5,4]
Output: 6
Explanation: [4,-1,2,1]
```

Commonly asked in:

* Amazon
* Microsoft
* LinkedIn

---

# Solution Patterns Overview

| Pattern                        | Idea                    | Time       | Space    | Best Use                   |
| ------------------------------ | ----------------------- | ---------- | -------- | -------------------------- |
| Brute Force                    | Try all subarrays       | O(n²)      | O(1)     | Learning                   |
| Prefix Sum                     | Use cumulative sums     | O(n²)      | O(n)     | Intermediate understanding |
| Dynamic Programming (Kadane’s) | Track best current sum  | O(n)       | O(1)     | Optimal                    |
| Divide and Conquer             | Split array recursively | O(n log n) | O(log n) | Advanced interviews        |

# 2. Prefix Sum Pattern

## Pattern Idea

Use prefix sums to calculate subarray sums faster.

Prefix formula:

Sum(i,j)=prefix[j+1]-prefix[i]

## Pattern Type

* Prefix Sum
* Range Sum Query

## Code

```python
def max_subarray(nums):
    # Code Here


nums = [-2,1,-3,4,-1,2,1,-5,4]
print(max_subarray(nums))
```

## Complexity

* Time: O(n²)
* Space: O(n)

## Advantage Over Brute Force

* Subarray sum becomes O(1)
* Avoids repeated summation

# 3. Dynamic Programming Pattern (Kadane’s Algorithm)

## Pattern Idea

At every position:

* Either extend previous subarray
* Or start new subarray

Transition:

current_sum = \max(nums[i],\ current_sum + nums[i])

Global answer:

maximum = \max(maximum,current_sum)

---

## Pattern Type

* Dynamic Programming
* Greedy + DP Hybrid

---

* Time: O(n)
* Space: O(1)

# 4. Divide and Conquer Pattern

## Pattern Idea

Maximum subarray is either:

1. Left half
2. Right half
3. Crossing middle

## Pattern Type

* Divide and Conquer
* Recursion

---

## Code

```python
def max_crossing_sum(nums, left, mid, right):

    left_sum = float('-inf')
    current = 0

    for i in range(mid, left - 1, -1):
        current += nums[i]
        left_sum = max(left_sum, current)

    right_sum = float('-inf')
    current = 0

    for i in range(mid + 1, right + 1):
        current += nums[i]
        right_sum = max(right_sum, current)

    return left_sum + right_sum


def divide(nums, left, right):

    if left == right:
        return nums[left]

    mid = (left + right) // 2

    left_max = divide(nums, left, mid)
    right_max = divide(nums, mid + 1, right)

    cross_max = max_crossing_sum(nums, left, mid, right)

    return max(left_max, right_max, cross_max)


def max_subarray(nums):
    return divide(nums, 0, len(nums) - 1)


nums = [-2,1,-3,4,-1,2,1,-5,4]
print(max_subarray(nums))
```

## Complexity

* Time: O(n log n)
* Space: O(log n)

# Optimized Solution Pattern

## Best Pattern → Kadane’s Algorithm

Why?

* Single traversal
* Constant space
* Most expected interview answer
* Elegant DP optimization

## Core Insight

If previous sum becomes negative:

* discard it
* start fresh from current element

Because:

negative\ prefix + current < current

---

# Final Optimized Code

```python
def max_subarray(nums):

    current_sum = nums[0]
    maximum = nums[0]

    for num in nums[1:]:

        current_sum = max(num,
                          current_sum + num)

        maximum = max(maximum,
                      current_sum)

    return maximum


nums = [-2,1,-3,4,-1,2,1,-5,4]

print(max_subarray(nums))
```
