# Minimum Window Substring

Find the **smallest substring** in string `s` that contains all characters of string `t`.

Example:

```text
Input:
s = "ADOBECODEBANC"
t = "ABC"

Output:
"BANC"
```

---

# Pattern Recognition

This problem is mainly solved using:

1. **Brute Force Enumeration**
2. **Sliding Window + Frequency Count** ← Optimized
3. **Sliding Window + Counter Optimization**

---

## Idea

Generate every substring and check whether it contains all characters of `t`.

# 2. Sliding Window Pattern (Standard Optimized)

## Main Idea

Use:

* `right` → expand window
* `left` → shrink window
* Frequency maps
* Track when all characters are matched

This is the core optimized approach.

---

# Visualization

```text
s = "ADOBECODEBANC"
t = "ABC"

Expand right →
ADOBEC

Valid window found

Shrink left →
DOBEC  ❌ missing A

Expand again →
CODEBA

Expand →
BANC ✅
```

---

# Core Logic

We maintain:

* `target_count`
* `window_count`
* `have`
* `need`

Where:

```text
need = number of unique chars in t
have = matched unique chars
```

When:

```text
have == need
```

window becomes valid.

---

# Time Complexity

```text
O(N)
```

Each character moves:

* once by right
* once by left

---

# Space Complexity

```text
O(256) or O(T)
```

# Pattern Comparison

| Pattern                | Idea                    | Time  | Space  | Interview Value |
| ---------------------- | ----------------------- | ----- | ------ | --------------- |
| Brute Force            | Generate all substrings | O(N³) | O(T)   | Basic           |
| Sliding Window         | Expand + shrink         | O(N)  | O(T)   | Very High       |
| Sliding Window + Array | Hash optimization       | O(N)  | O(128) | High            |

---

# Best Optimized Pattern

## Sliding Window

Why?

* Linear time
* Most expected in interviews
* Standard template for:

  * smallest substring
  * longest substring
  * anagrams
  * distinct characters
  * window problems

---

# Important Interview Insight

This problem teaches the master sliding-window template:

```python
for right in range(n):

    add current char

    while window is valid:

        update answer

        shrink from left
```

This same pattern is used in:

* Longest Repeating Character Replacement
* Permutation in String
* Find All Anagrams
* Longest Substring with K Distinct Characters
* Smallest Window Problems

# Edge Cases

## Empty strings

```python
s = ""
t = "ABC"
```

Output:

```python
""
```

---

## No valid substring

```python
s = "HELLO"
t = "XYZ"
```

Output:

```python
""
```

---

## Repeated characters in target

```python
s = "AAABBC"
t = "AABC"
```

Need:

* 2 A's
* 1 B
* 1 C

Sliding window handles this correctly using frequencies.
