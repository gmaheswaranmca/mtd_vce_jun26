# Group Anagrams

Group strings that are anagrams of each other.

Example:

```text
Input: ["eat","tea","tan","ate","nat","bat"]

Output:
[
  ["eat","tea","ate"],
  ["tan","nat"],
  ["bat"]
]
```

---

# Main Idea

Two words are anagrams if:

* same characters
* same frequencies

Examples:

```text
eat -> aet
tea -> aet
ate -> aet
```

All belong to same group.

---

# Patterns Used

## 1. Brute Force Pattern

### Idea

Compare every string with every other string.

If sorted versions are same → anagram.

### Pattern

* visited array
* nested loops
* compare sorted strings

---

## 2. HashMap + Sorted String Pattern

### Idea

Use sorted string as key.

```text
eat -> aet
tea -> aet
```

Store all words with same sorted key.

### Pattern

* hashing
* frequency grouping
* dictionary/list

Most common interview solution.

---

## 3. HashMap + Character Count Pattern (Optimized)

### Idea

Instead of sorting:

Count character frequencies.

```text
eat -> [1,0,0,0,1,....,1]
tea -> same count array
```

Use tuple(count) as hashmap key.

### Pattern

* frequency counting
* fixed-size array
* hashing

Better than sorting.

---

# Pattern Comparison

| Pattern               | Idea                   | Time            | Space | Interview Value     |
| --------------------- | ---------------------- | --------------- | ----- | ------------------- |
| Brute Force           | Compare all pairs      | O(n² * k log k) | O(n)  | Basic understanding |
| HashMap + Sorting     | Sorted word as key     | O(n * k log k)  | O(n)  | Standard solution   |
| HashMap + Count Array | Frequency tuple as key | O(n * k)        | O(n)  | Optimized           |

Where:

* `n` = number of strings
* `k` = average string length

# Why tuple(count)?

List cannot be hashmap key.

```python
[1,0,2] ❌
(1,0,2) ✅
```

Tuple is immutable.

---

# Complexity Analysis

## Sorting Pattern

For every word:

```text
sorting cost = O(k log k)
```

Total:

```text
O(n * k log k)
```

---

## Frequency Count Pattern

For every word:

```text
count characters = O(k)
```

Total:

```text
O(n * k)
```

Faster for large strings.

---

# Which Pattern is Best?

## Recommended

### HashMap + Character Frequency

Because:

* avoids sorting
* linear processing
* scalable
* strong hashing pattern
